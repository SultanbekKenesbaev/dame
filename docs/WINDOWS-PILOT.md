# Windows pilot checklist

## Режимы Windows

- **Обычный режим по умолчанию:** поддерживает Windows 10/11 Pro, Enterprise, Education и IoT Enterprise. Сначала загружается Explorer и рабочий стол, затем DailyGate автоматически открывается обычным окном. Пользователь может свернуть, развернуть или закрыть приложение; остальные программы и мониторы не блокируются.
- **Необязательный полный kiosk:** включается только параметром `-KioskMode` на Windows Enterprise, Education или IoT Enterprise. Shell Launcher заменяет Explorer, действует принудительный выход в `04:00`, а стандартный пользователь не может закрыть тест обычными средствами.

В обычном режиме принудительный выход из Windows в `04:00` отключён. Если клиент остаётся запущенным, он сам покажет тест нового рабочего дня. Если пользователь закрыл программу, она снова автоматически откроется при следующем входе в Windows.

## Перед установкой

- Проверить редакцию Windows. Обычный режим работает в целевых редакциях Windows 10/11; полный kiosk требует Enterprise/Education/IoT.
- Создать и протестировать отдельного локального Windows-администратора, не затрагиваемого kiosk-политиками.
- Создать полную резервную копию или recovery image компьютера.
- Перенести нужные рабочие файлы и настройки в будущий kiosk-профиль.
- Проверить доступность API по HTTPS и корректность системного времени.
- Получить code-signing сертификат и собирать через `build-windows.ps1 -CertificateThumbprint ...`; скрипт подписывает и проверяет службу, клиент и MSI. Неподписанный prototype нельзя массово разворачивать.

## Установка

1. В админке создать сотрудника и назначить ему группу с опубликованным правилом теста.
2. Создать для сотрудника 15-минутный код подключения устройства.
3. Запустить `DailyGate.Setup.msi` двойным кликом и подтвердить запрос Windows на установку.
4. Дождаться автоматического открытия DailyGate. Если окно было закрыто, открыть DailyGate из меню «Пуск».
5. Ввести код подключения прямо в окне DailyGate и нажать «Подключить этот компьютер».
6. После успешного подключения войти под логином сотрудника и временным паролем; при первом онлайн-входе задать новый пароль.
7. Убедиться, что окно имеет отметку `ОБЫЧНЫЙ РЕЖИМ · БЕЗ БЛОКИРОВКИ` и его можно свернуть, развернуть или закрыть.
8. Выйти из Windows и войти снова: DailyGate должен открыться автоматически без повторного кода подключения.
9. Перезагрузить ПК в те же рабочие сутки: повторный тест не должен появиться.

PowerShell, изменение execution policy и `Provision-Device.ps1` для стандартной установки не используются. Скрипт оставлен только для отдельной технической настройки полного kiosk-режима.

## Обязательные испытания

- Отключить интернет и пройти заранее загруженный тест; после сети проверить синхронизацию.
- Остановить API не дольше семи дней и проверить offline lease.
- Проверить expiry таймера, частичные ответы и статус `TimedOut`.
- В обычном режиме проверить, что `Alt+F4`, кнопка закрытия и сворачивание работают, а рабочий стол остаётся доступным.
- В необязательном полном kiosk закрыть клиент через Alt+F4 и Task Manager под kiosk-профилем; доступ не должен появиться.
- В обычном режиме проверить, что второй монитор не закрывается shield-окном.
- Сгенерировать emergency code, применить один раз и проверить аудит/replay rejection.
- Сдвинуть часы назад в тестовой среде и проверить security event.
- В тестовый день временно изменить границу рабочего дня на ближайшее время и проверить предупреждения/logoff.
- Зайти отдельным локальным администратором и выполнить recovery.

## Удаление автозапуска или kiosk

1. Войти отдельным локальным Windows-администратором.
2. Запустить elevated PowerShell:

```powershell
& "$env:ProgramFiles\DailyGate\Tools\Remove-Kiosk.ps1"
```

3. Перезагрузить Windows.
4. После появления обычного Explorer удалить DailyGate через Installed Apps. В обычном режиме скрипт удаляет задачу автозапуска клиента.

Если Windows не позволяет войти локальным администратором, использовать Windows Recovery Environment и заранее подготовленную recovery-процедуру организации.
